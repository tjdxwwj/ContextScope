## 2) GET /plugins/contextscope/api/requests

按过滤条件返回 LLM 请求/响应事件列表。

Query 参数：
- `sessionId`（可选）
- `runId`（可选）
- `provider`（可选）
- `model`（可选）
- `startTime`（可选，毫秒时间戳）
- `endTime`（可选，毫秒时间戳）
- `limit`（可选，默认 100）
- `offset`（可选，默认 0）

响应结构：
- `requests`: `RequestData[]`
- `total`: 当前返回条数
- `filters`: 实际生效的过滤条件

说明：
- 默认不截断 `prompt`（`capture.maxPromptLength` 默认值为 `0`，表示不限制长度）。如需限制长度，可在插件配置里设置 `capture.maxPromptLength`（单位：字符）。

## 10) GET /plugins/contextscope/api/chain/{runId}

**新增接口** - 返回完整的调用链上下文（无分析逻辑）

### 请求参数

**Path 参数：**
- `runId`（必填）- 调用 ID

**Query 参数：**
- `limit`（可选，默认 100）- 每页数量
- `offset`（可选，默认 0）- 偏移量

### 响应结构

```typescript
interface ChainResponse {
  runId: string;
  sessionId: string;
  provider: string;
  model: string;
  startTime: number;
  endTime?: number;
  duration?: number;
  
  pagination: {
    limit: number;
    offset: number;
    total: number;
    hasMore: boolean;
  };
  
  chain: ChainItem[];  // 按时间倒序排列（最近的在前）
  
  stats: {
    totalItems: number;
    inputCount: number;
    outputCount: number;
    toolCallCount: number;
    subagentCount: number;
    totalTokens: number;
  };
}

interface ChainItem {
  id: string;
  runId: string;
  parentRunId?: string;  // 子 Agent 关联到父调用
  type: 'input' | 'output' | 'tool_call' | 'tool_result' | 'subagent_spawn' | 'subagent_result';
  timestamp: number;
  duration?: number;
  
  input?: {
    prompt?: string;
    systemPrompt?: string;
    historyMessages?: any[];  // 完整内容，不截断
    params?: any;
    task?: string;
  };
  
  output?: {
    text?: string;
    assistantTexts?: string[];  // 完整内容，不截断
    result?: any;               // 完整内容，不截断
    outcome?: string;
  };
  
  usage?: {
    input: number;
    output: number;
    total: number;
  };
  
  metadata?: {
    provider?: string;
    model?: string;
    toolName?: string;
    agentId?: string;
    status?: 'success' | 'error' | 'pending';
    error?: string;
  };
}
```

### 设计原则

1. **扁平化结构** - 所有项目按时间倒序排列，不区分层级
2. **统一字段** - 所有项目都有 `id`, `runId`, `type`, `timestamp`
3. **类型明确** - `type` 字段清晰标识项目类型
4. **无分析逻辑** - 不包含 heatmap、insights 等分析数据
5. **完整上下文** - 所有内容不截断
6. **分页支持** - limit/offset 控制，响应包含 total 和 hasMore

### 与旧接口对比

| 特性 | 旧接口 `/api/analysis` | 新接口 `/api/chain` |
|------|----------------------|-------------------|
| 分析逻辑 | ✅ 包含 | ❌ 不包含 |
| Heatmap | ✅ 包含 | ❌ 不包含 |
| Insights | ✅ 包含 | ❌ 不包含 |
| 原始数据 | ⚠️ 部分 | ✅ 完整 |
| Tools | ⚠️ 聚合 | ✅ 详细（call + result） |
| Subagents | ⚠️ 聚合 | ✅ 详细（spawn + result） |
| 父子关联 | ❌ | ✅ parentRunId |
| 分页 | ❌ | ✅ limit/offset |
| 排序 | 升序 | 降序（最近的在前） |

## 11) GET /plugins/contextscope/api/context

**上下文分布接口** - 返回单次调用的完整上下文分布信息

### 接口说明

返回单次 API 调用的完整上下文分布，包括：
- ✅ System Prompt（完整内容，不截断）
- ✅ User Prompt（完整内容，不截断）
- ✅ History Messages（完整历史对话）
- ✅ Token Distribution（各部分 token 占比）
- ✅ Model Info（模型信息、context window、预估成本）
- ✅ Tool Calls（工具调用记录）
- ✅ Subagent Links（子 Agent 关联）

**适用场景**：调试 Prompt、分析 Token 消耗、上下文优化、成本估算

### 请求参数

**Query 参数：**
- `runId`（必填）- 调用 ID

### 示例请求
```bash
GET /plugins/contextscope/api/context?runId=test-context-1773250903869
```
