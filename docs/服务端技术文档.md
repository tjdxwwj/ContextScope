# ContextScope（OpenClaw 插件）调用链与 Token 统计技术文档（当前实现版）

插件代码仓库：`D:\code\request-analyzer`  
OpenClaw 主项目：`D:\code\openclaw`

## 1. 目标

- 采集每次 Agent 执行（run）的 LLM 请求/响应上下文与 token usage
- 采集每次工具调用（tool call）的参数、结果、耗时
- 建立主/子 Agent（subagent）协作的关联边（spawn、send、ended）
- 将采集结果落到本地持久化存储（当前：JSON 单文件），并通过 HTTP API 提供查询

## 2. 关键标识

- `runId`：一次 Agent 调用（一个 turn）的稳定标识。一个 `runId` 可能包含多次 LLM 调用（例如工具回合）。
- `sessionId`：短期会话 UUID（`/new`、`/reset` 后会变）。适合做短期聚合与 UI 展示。
- `sessionKey`：稳定会话键（跨 reset 更稳定）。适合做长期归档/关联。
- `toolCallId`：provider 侧的工具调用标识（可选）。在同一 run 内用于区分/配对工具调用。

- `historyMessages`：当次请求真正送入模型的 messages 列表（包含此前 tool_result 等）

- 目的：补齐子任务的结束态（success/error/timeout/aborted）
- 依据：`subagent_ended` 携带 `runId/outcome/error/endedAt`

## 6. 数据治理与安全

采集开关（建议用于降低敏感信息落盘风险）：
- `capture.includeSystemPrompts`：是否保存 systemPrompt（默认开）
- `capture.includeMessageHistory`：是否保存 historyMessages（默认开）

脱敏：
- `capture.anonymizeContent`：对 `prompt/systemPrompt/assistantTexts/historyMessages` 执行字符串级脱敏（见 [service.ts](file:///d:/code/request-analyzer/src/service.ts#L435-L520)）

保留策略：
- `storage.retentionDays` 与 `storage.maxRequests` 会同时约束 `requests/toolCalls/subagentLinks`（见 [storage.ts](file:///d:/code/request-analyzer/src/storage.ts#L392-L406)）
